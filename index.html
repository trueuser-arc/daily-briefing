ChromaDB Pre-Auth RCE (CVE-2026-45829) — Max-Severity Flaw Threatens AI App Infrastructure

A max-severity vulnerability in ChromaDB's Python FastAPI server lets unauthenticated attackers execute arbitrary code by supplying a malicious HuggingFace model reference as an embedding function. The root cause is two independent failures: the server trusts client-supplied model identifiers without restriction and acts on that trust before authenticating the request. Introduced in version 1.0.0 and still unpatched as of 1.5.8, the flaw can leak API keys, environment variables, mounted secrets, and all files on disk. Of internet-exposed ChromaDB instances, 73% are running vulnerable versions — and ChromaDB is the most popular open-source vector database for RAG and AI agent workflows.

If you run ChromaDB's Python server, switch to the Rust frontend immediately or ensure all traffic passes through an authenticating reverse proxy. Never expose the Python FastAPI server directly to the internet — the pre-auth RCE means there's no safe configuration until a patch ships.

Exim "Dead.Letter" RCE (CVE-2026-45185) — CVSS 9.8 Unauthenticated Mail Server Takeover

A critical use-after-free vulnerability dubbed "Dead.Letter" in Exim's BDAT parsing over GnuTLS lets unauthenticated attackers achieve remote code execution on mail servers. During TLS shutdown, Exim frees its transfer buffer but a nested BDAT receive wrapper still writes a single character into the freed region — corrupting the allocator's internal metadata. Discovered by XBOW and disclosed May 12, the bug affects Exim 4.97 through 4.99.2 on GnuTLS builds with STARTTLS and CHUNKING advertised. OpenSSL builds are not affected. With Exim handling roughly 30% of the internet's email traffic, the blast radius is substantial — patch to 4.99.3 immediately or disable CHUNKING as a stopgap.
