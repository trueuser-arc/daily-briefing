Check Point VPN Zero-Day CVE-2026-50751 (CVSS 9.3) Exploited by Qilin Ransomware — CISA Orders Emergency Patch

A critical authentication bypass in Check Point Remote Access VPN, Mobile Access, and Spark Firewall products has been under active exploitation since at least May 7, with Qilin ransomware affiliates confirmed as one operator. CVE-2026-50751 affects deployments using the deprecated IKEv1 protocol where gateways accept legacy clients without requiring machine certificates — the flaw lets an unauthenticated attacker establish a full VPN session without valid credentials. Check Point characterized the campaign as affecting "several dozen organizations," but the threat actor infrastructure is simultaneously exploiting VPN vulnerabilities across Palo Alto, Fortinet, and F5 products, suggesting a coordinated perimeter-targeting campaign. CISA added the vulnerability to its KEV catalog and gave federal agencies three days to patch.

If your organization runs Check Point VPN with IKEv1 enabled, apply the hotfix immediately and audit VPN session logs back to May 7. The same threat actors are chaining VPN vulnerabilities across multiple vendors — this is a perimeter-wide campaign, not a single-product issue.

Chrome V8 Zero-Day CVE-2026-11645 Exploited in the Wild — Fifth Chrome Zero-Day of 2026, CISA Adds to KEV

Google patched CVE-2026-11645 on June 8 — an out-of-bounds memory access in V8 (CVSS 8.8) that allowed remote code execution inside Chrome's sandbox via a crafted HTML page. The flaw was discovered by researcher "303f06e3" on April 27 and earned a $55,000 bug bounty, but Google confirmed an exploit existed in the wild before the patch shipped. This is the fifth Chrome zero-day exploited in 2026, following CVE-2026-2441, CVE-2026-3909, CVE-2026-3910, and CVE-2026-5281 — a pace that exceeds 2025's full-year count by June. CISA added it to the KEV catalog with a federal patching deadline. For iOS developers, the V8 vulnerability doesn't directly affect WebKit-based browsers on Apple platforms, but any Electron apps or server-side V8 usage (Node.js, Deno, Bun) in your toolchain should be updated to Chrome 149.0.7827.103 or later.

FortiBleed Compromised Count Reaches 86,644 — CISA Issues Dedicated Hardening Alert for Fortinet Devices

The FortiBleed credential-harvesting campaign first reported last week at 73,932 compromised Fortinet firewalls has grown to 86,644 confirmed working credentials across 194 countries, prompting CISA to issue a dedicated hardening alert on June 18. The attackers are running an automated feedback loop: scan for exposed FortiGate devices, try curated password lists, then use compromised firewalls as listening posts to intercept additional SSL VPN credentials flowing through the device — which get fed back into the scanner. CISA's guidance urges organizations to terminate all active VPN sessions, reset credentials using PBKDF2 hashing, enable phishing-resistant MFA, and lock down management interfaces. Major organizations including Chevron, Samsung, Foxconn, and multiple government agencies appear in the leaked database.

Apple Open-Sources Foundation Models Framework at WWDC26 — Adds Claude, Gemini Support and Free Private Cloud Compute for Small Developers

At WWDC26, Apple announced it will open-source the Foundation Models framework — the on-device AI layer powering Apple Intelligence — later this summer, alongside two companion implementations: CoreAILanguageModel and MLXLanguageModel for running local models on Apple Silicon. The framework now supports third-party models like Claude and Gemini through a universal LanguageModel protocol, letting developers swap providers without rewriting integration code. New image input support enables on-device visual tasks (receipt extraction, photo captioning, UI classification) with zero cloud round-trip. Most significantly for indie developers, Apple is offering free access to its cloud models running on Private Cloud Compute for apps with fewer than two million first-time App Store downloads — effectively eliminating the AI inference cost barrier for the vast majority of iOS apps.

The free Private Cloud Compute tier for sub-2M-download apps is the buried lede. Combined with the open-source LanguageModel protocol supporting Claude and Gemini, Apple is positioning its platform as the zero-cost on-ramp for AI-powered iOS apps. Start prototyping with the Foundation Models framework now — the open-source release this summer will let you evaluate on-device vs. cloud trade-offs before iOS 27 ships.
