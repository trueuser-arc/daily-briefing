This is a textbook case of AI agents being given too much authority without adequate identity verification. Exim "Dead.Letter" CVE-2026-45185 — CVSS 9.8 Unauthenticated RCE Threatens Millions of GnuTLS-Based Mail Servers

CVE-2026-45185 ("Dead.Letter") is a critical use-after-free vulnerability in Exim versions 4.97 through 4.99.2 that enables unauthenticated remote code execution on GnuTLS-based builds. The flaw triggers when Exim frees its TLS transfer buffer during shutdown but continues using stale callback references during BDAT chunked SMTP delivery — an attacker can corrupt the heap by sending cleartext bytes after a TLS close_notify. Exploitation requires only a standard SMTP connection with CHUNKING enabled, which is the default. Exim is the world's most widely deployed MTA, and GnuTLS is the default TLS library on Debian-based systems — the overlap is massive. XBOW researcher Federico Kirschbaum discovered the flaw using AI-assisted security research tooling, making it one of the first major CVEs attributed to AI-aided vulnerability discovery. Exim 4.99.3 patches the issue.

SolarWinds Serv-U CVE-2026-28318 Added to CISA KEV — Unauthenticated DoS Crashes File Transfer Servers in the Wild

CISA added CVE-2026-28318 (CVSS 8.8) to the Known Exploited Vulnerabilities catalog on June 5, with a federal remediation deadline of June 19. The flaw resides in Serv-U's unblockClientOnKey() function: an unauthenticated attacker can crash the service with a single crafted HTTP POST request using a Content-Encoding: deflate header. While rated as denial-of-service, the real-world impact is significant — Shodan shows over 12,000 Serv-U instances exposed to the internet, many serving as enterprise file transfer gateways. SolarWinds patched it in version 15.5.4 Hotfix 1, but admins who upgraded to 15.5.4 without applying the hotfix remain vulnerable.

Chinese APT UNC5221 Hid Inside Microsoft 365 Networks for 18 Months — Brickstorm Backdoor and New Malware Disclosed

Researchers disclosed that Chinese espionage group UNC5221 (also tracked as VerdantBamboo) maintained undetected access to multiple U.S. organizations' Microsoft 365 environments for at least 18 months before discovery. The group exploited zero-day vulnerabilities in edge network appliances to gain initial access, then deployed the Brickstorm backdoor alongside two previously undocumented tools: Plenet (a cross-platform .NET backdoor providing remote command execution, file management, and interactive shell access) and AgentPSD. UNC5221's targets include legal services firms, SaaS providers, business process outsourcers, and technology companies — a pattern consistent with long-term intelligence collection rather than financially motivated attacks.

Hola Browser Supply Chain Attack — Windows Installer Delivered Monero Miner to Users

Sophos researchers discovered that the Windows version of Hola Browser (version 1.251.91.0) was compromised in a supply chain attack that bundled a Monero cryptocurrency miner disguised as me.exe. The miner adds a Windows Defender exclusion rule, copies itself to Program Files as HolaMonitorService.exe, creates an auto-starting Windows service, and activates when the system is idle — a stealthy approach designed to avoid user detection. The compromise was caught during routine AppEsteem certification testing. Hola says approximately 0.1% of its users were affected and has since rebuilt its distribution pipeline. The incident follows a pattern of smaller browser vendors becoming supply chain targets — similar to the DAEMON Tools trojanized installers disclosed two weeks ago.

June Patch Tuesday Preview — Exchange Fix Expected Tuesday, Critical Secure Boot Deadline Looms June 26

June's Patch Tuesday arrives on June 9 with expected fixes for the actively exploited Exchange OWA zero-day (CVE-2026-42897) that has lacked a permanent patch since disclosure. More critically, this is the final opportunity to deploy Secure Boot dbx (certificate revocation) updates before Microsoft's absolute deadline of June 26 — after which unpatched systems will no longer accept new Secure Boot policies. Historical data shows 2–5% of systems experience boot failures after dbx deployment, translating to 20–50 machines per 1,000 requiring emergency recovery. Enterprise admins should stage the dbx update in test environments this week and have recovery media ready before the Tuesday rollout.

Brave Launches Origin — $59.99 Paid Browser Strips Crypto, AI, and Rewards for Privacy Purists

Brave Software released Brave Origin, a paid variant that compile-out removes 13 features — including Leo AI, Brave Rewards, Brave Wallet, VPN promotions, Brave News, Brave Talk, Tor windows, and sponsored images — while retaining the core Brave Shields ad-blocking and tracker-protection engine. Priced at $59.99 one-time on Windows and macOS (free on Linux, coming to iOS/Android in v1.91), Origin represents an unusual business model experiment: charging users to get less software. The launch drew mixed reactions — critics argue Brave is monetizing the removal of features many users already disabled via settings. Supporters counter that a compile-out build, not just toggled-off features, is meaningfully different: smaller binary, reduced attack surface, and no risk of features re-enabling via updates.

Trump Signs AI Executive Order — Voluntary 30-Day Frontier Model Review, NSA Sets Classification Threshold

On June 2, President Trump signed "Promoting Advanced Artificial Intelligence Innovation and Security," an executive order directing federal agencies to establish a framework for secure deployment of frontier AI models. The headline provision: AI companies are asked to voluntarily submit their most powerful models for government testing up to 30 days before public release — but the order explicitly states this is not a mandatory licensing, pre-clearance, or permitting requirement. The NSA Director will set the threshold for which models qualify as "covered frontier models" through a classified benchmarking process assessing advanced cyber capabilities. The order also establishes an AI cybersecurity clearinghouse for vulnerability remediation at scale. The emphasis on voluntary compliance represents a middle path between the security establishment's push for mandatory review and the administration's anti-regulatory stance.
