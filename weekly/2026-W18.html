GitHub RCE (CVE-2026-3854): Single git push Gave Cross-Tenant Access to Millions of Repos — 88% of Enterprise Servers Still Unpatched

Wiz Research publicly disclosed CVE-2026-3854 (CVSS 8.7), a command injection flaw in GitHub's internal git infrastructure that allowed any authenticated user with push access to execute arbitrary commands via crafted push option values. Because user-supplied X-Stat header values were not sanitized, an attacker could inject metadata fields that override security-critical configuration and bypass sandboxing. Due to GitHub's multi-tenant architecture, achieving code execution on a shared storage node exposed millions of public and private repositories belonging to other organizations. GitHub deployed a fix to github.com within two hours of Wiz's March 4 report and found no evidence of exploitation, but Wiz's scan data shows 88% of internet-reachable GitHub Enterprise Server instances remain unpatched.

Upgrade all GHES instances to 3.19.3 (or your branch's patched version) immediately. Any authenticated user — including bots and CI tokens — could have exploited this with a stock git client.
