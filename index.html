Sentry SAML SSO Flaw Enables Zero-Click Account Takeover (CVE-2026-42354)

A critical vulnerability in Sentry's SAML SSO implementation lets an attacker spin up a malicious Identity Provider, send a crafted SAML assertion with any victim's email, and instantly link the victim's account to the attacker's session — no user interaction or password required. The flaw stems from Sentry trusting cross-organization IdP assertions without validating origin. Any self-hosted Sentry instance with SAML enabled is vulnerable. Given Sentry's ubiquity in developer workflows, this grants access to error tracking data, source maps, and potentially production secrets stored in breadcrumbs.
