Bluekit PhaaS Adopts Browser-in-the-Middle Attack — Bypasses All Traditional MFA for Microsoft Login Theft

The Bluekit phishing-as-a-service platform has evolved from adversary-in-the-middle to a more sophisticated browser-in-the-middle (BitM) architecture, with nearly 70 new hostnames identified in the past week alone. Instead of proxying traffic like Evilginx, Bluekit loads the legitimate Microsoft login page inside an attacker-controlled browser and uses the open-source 'rrweb' library to serialize the DOM, streaming it to victims over a WebSocket connection. The result: victims interact with what appears to be the real login page while Bluekit captures credentials, session tokens, and MFA approvals in real time. Anti-analysis defenses include randomized CSS filters, browser fingerprinting, WebRTC IP mismatch detection, and custom CAPTCHAs. SMS codes, authenticator apps, and push approvals provide zero protection against this architecture.
