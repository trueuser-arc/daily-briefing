CVE-2026-47291: Windows HTTP.sys RCE (CVSS 9.8) — Integer Overflow Enables Unauthenticated Remote Code Execution on Any Windows Server

Buried alongside the headline-grabbing wormable TCP/IP flaw (CVE-2026-45657) in June's record Patch Tuesday is CVE-2026-47291, a CVSS 9.8 remote code execution vulnerability in Windows HTTP.sys — the kernel-mode driver that sits beneath IIS and any application using the Windows HTTP Server API. An integer overflow and heap-based buffer overflow allow unauthenticated remote attackers to execute code by sending specially crafted packets to the HTTP Protocol Stack. HTTP.sys is one of the most widely exposed Windows components, processing traffic for web servers, WinRM, SSDP, and numerous internal services. Servers with MaxRequestBytes set above the default 16 KB are directly exploitable. Unlike CVE-2026-45657, this flaw hasn't received the "wormable" designation, but its attack surface is arguably broader given how many services rely on HTTP.sys.

Prioritize CVE-2026-47291 alongside CVE-2026-45657. HTTP.sys is ubiquitous — check MaxRequestBytes on every Windows server and apply the June patch immediately. Servers with non-default MaxRequestBytes values above 65534 bytes are directly exposed.
